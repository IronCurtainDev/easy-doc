# Contributing to EasyDoc

Thank you for considering contributing to EasyDoc! We welcome contributions from everyone.

## Code of Conduct

Please be respectful and considerate of others when contributing to this project.

## How to Contribute

1.  **Fork the Repository**: Create a fork of this repository to your own GitHub account.
2.  **Create a Branch**: Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/my-new-feature
    ```
3.  **Make Changes**: Write your code and tests. Ensure you follow the existing coding style (PSR-12).
4.  **Run Tests**: Run the automated tests to ensure everything is working correctly.
    ```bash
    composer test
    ```
5.  **Commit Changes**: Commit your changes with a clear and descriptive message.
    ```bash
    git commit -m "Add feature: my new feature"
    ```
6.  **Push to Fork**: Push your branch to your forked repository.
    ```bash
    git push origin feature/my-new-feature
    ```
7.  **Submit Pull Request**: Open a Pull Request from your branch to the main repository.

## Coding Standards

-   We follow **PSR-12** coding standards.
-   All new features should be accompanied by tests.
-   Ensure `declare(strict_types=1);` is present in all PHP files.

## Reporting Bugs

If you find a bug, please open an issue on GitHub with a detailed description, creating a reproduction steps if possible.
